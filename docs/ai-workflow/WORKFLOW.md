# AI Workflow

## Full Workflow

1. Project Context Loader
2. Project Integrity Check
3. Diagnosis
4. Review
5. Scope Approval
6. Chief Architect -> Codex Handoff
7. Codex Implementation
8. Chief Architect Verification
9. User Acceptance
10. Commit
11. Push
12. Documentation Update
13. Documentation Review
14. Milestone Close
15. Session Close

## Meaning Of The Flow

Project Context Loader establishes the required project context.

Project Integrity Check verifies that documentation and repository state are ready for work.

Diagnosis and Review happen before implementation.

Codex implements only after scope approval and handoff.

Documentation is updated only after implementation and repository actions are confirmed.

## Important Distinction

Milestone Close is not the same as Session Close.

Milestone Close means the approved scope for the milestone is complete, verified, documented, and ready for repository closure.

Session Close means the current collaboration session is finished and the next session can resume safely from updated documentation.

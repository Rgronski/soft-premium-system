# ENG-000 Constitution

## Purpose

This document defines the permanent operating principles of ENG-000 - AI Development Platform.

These rules remain stable unless they are explicitly superseded by approved documentation governance.

## Constitutional Rules

1. Documentation is the Single Source of Truth.
2. SPS is the primary interface to the AI Development Platform.
3. ChatGPT acts as Chief Architect.
4. Chief Architect does not implement production code.
5. Codex acts as Coder / Executor.
6. Codex implements approved scope only.
7. GitHub is the auditable repository state.
8. Every workflow starts with PCL.
9. Every implementation requires Project Integrity Check.
10. Automate processes, not architectural decisions.
11. Milestone Close is not Session Close.
12. No commit or push without Product Owner approval.
13. Stop on documentation inconsistency.

## Interpretation

These rules define workflow authority, execution boundaries, and the relationship between documentation, implementation, and repository state.

They exist to keep SPS, ChatGPT, Codex, and GitHub aligned under one operational model.

## Practical Meaning

The constitution means:

* workflow starts from verified context, not assumptions,
* implementation follows approved scope rather than independent tool decisions,
* documentation state must remain coherent before workflow continues,
* automation should strengthen execution discipline without replacing architectural judgment.

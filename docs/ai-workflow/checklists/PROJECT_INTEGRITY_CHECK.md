# Project Integrity Check

Use this checklist before every implementation session.

## Current State ↔ Session State

* Do `08_CURRENT_STATE.md` and `10_SESSION_STATE.md` agree on the latest completed milestone?
* Do they agree on the current milestone?
* Do they agree on the next milestone or next step?

## Changelog ↔ Current State

* Does `09_CHANGELOG.md` include the latest completed milestone?
* Does `08_CURRENT_STATE.md` match that completed milestone?

## Milestone Status

* Is the previous milestone actually complete?
* Is the next milestone clearly identified?
* Is there any milestone ambiguity that must be resolved before implementation?

## Repo Readiness

* Is the repository state known?
* Is there any unreviewed work that could affect the next milestone?
* Is the implementation scope safe to begin?

## Documentation Consistency

* Is documentation terminology consistent?
* Is Source of Truth ownership still clear?
* Are workflow, session, and milestone states aligned?

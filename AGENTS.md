* Documentation is the Single Source of Truth.
* Codex is the Implementation Engine.
* Always perform diagnosis before editing files.
* Use one minimal patch with one clear intention.
* Do not refactor, reorder, or clean up unrelated code.
* Do not change files outside the approved scope.
* Stop immediately if implementation needs an extra file or an architectural decision.
* Deliver a final report with repository state before change, diagnosis, changed files, diff summary, verification results, `git diff --check`, `git diff --stat`, `git status --short`, intentional non-changes, commit/push status, and blockers or contract deviations.
* Do not commit or push without explicit Product Owner approval.
